# UTMS Playwright Test Strategy

## Scope and source of truth

This strategy tests the behavior that exists in this checkout. `apps/web` is a React/Vite application whose business domains are currently backed by browser-persistent and module-memory services. The Online API Console is the concrete HTTP backend at `apps/api/src/modules/api-console/infrastructure/http/api-console-server.cjs`, with file-backed persistence and `x-utms-context` authorization. PostgreSQL, Redis, the worker, and `apps/playwright-runner` are not silently treated as integrations for features that do not call them.

## Architecture

| Layer | Location | Runner | Isolation |
| --- | --- | --- | --- |
| Structural decision tests | `tests/structural` | Playwright Test + direct module imports | No network; deterministic seed |
| API integration and security | `apps/api/test` | `APIRequestContext` | `NODE_ENV=test`, reset endpoint, file store |
| Browser smoke/E2E/system | `tests/{smoke,e2e,system}` | Chromium project | Dedicated API/web processes |
| Accessibility | `tests/accessibility` | Chromium + `@axe-core/playwright` | RTL, Persian locale |
| Compatibility | `tests/compatibility` | Chromium, Firefox, WebKit | Same critical scenario per engine |
| Performance | `tests/performance` | APIRequestContext | Bounded local samples; JSON baseline |
| Reliability/regression/UAT | respective folders | Playwright projects | Repeatable fixtures and reset |

The repository's product-level `apps/playwright-runner` remains a product capability. It is not the QA harness. Docker uses `mcr.microsoft.com/playwright:v1.55.0-noble` as the separate `qa-tests` service.

## Determinism and safety

- `UTMS_TEST_SEED` defaults to `20260715`; generated inputs log that seed.
- No test uses `page.waitForTimeout` or an arbitrary sleep. Readiness uses health URLs and Playwright auto-waiting.
- Each API fixture resets the test-only file store before use. The reset route is 404 unless `NODE_ENV=test`.
- Context headers are generated by `tests/helpers/api-context.ts`; secrets are redacted by `tests/helpers/redaction.ts` before artifact attachment.
- Tests use only `example.com`, the isolated API, and the isolated web service. SSRF assertions verify that blocked destinations are not transmitted.

## Traceability and techniques

Every executable test has a stable `UTMS-...` ID and annotations for requirement, feature, level, type, ISO 29119-4 technique, role, scope, data, expected result, and risk. The technique-to-test mapping is maintained in `docs/testing/TEST_COVERAGE_MATRIX.md` and the catalog in `docs/testing/TEST_CASE_CATALOG.md`.

Structural evidence is kept separate from black-box evidence. E2E tests demonstrate user-visible behavior; they do not claim statement, branch, or MCDC coverage without direct module tests and `c8` output.

## Browser matrix

Chromium runs smoke, E2E, system, accessibility, regression, UAT, and performance suites. The compatibility project runs the critical navigation scenario on Chromium, Firefox, and WebKit. Mobile/RTL variants are represented by project configuration and the RTL assertions; heavier suites remain Chromium-only to keep pull-request feedback bounded.

## Reports and maintenance

Playwright writes HTML, JUnit, and JSON reports to ignored artifact paths. Traces are collected on first retry, screenshots on failure, and videos on failure. Performance samples are written to `artifacts/tests/performance`. Known gaps are explicit in `docs/testing/KNOWN_TEST_GAPS.md`; no gap is silently skipped.

## Local commands

```text
npm ci
npx playwright install chromium firefox webkit
npm run test:stack:up       # Docker-only isolated stack
npm run test:stack:wait
npm run test:smoke
npm run test:integration
npm run test:security
npm run test:e2e
npm run test:system
npm run test:accessibility
npm run test:compatibility
npm run test:structural
npm run test:coverage
npm run test:all
npm run test:stack:down
```

For a local non-Docker run, Playwright starts the test API and Vite automatically. Set `UTMS_CHROME_EXECUTABLE_PATH` when using a system Chromium installation.
